# Edit-System v0.2.0

This release makes the block-based edit workflow production-ready: reviews inbox, author UX, inline formatting, Typst PDF export, and near-realtime collaboration.

### Features

- **Reviews inbox:** `/reviews` lists pending suggestions for scope leads and your own submissions; dashboard and sidebar badge reflect the lead queue count.
- **Author UX:** “Suggest change” from the published block view; suggestions panel supports headings, paragraphs, code, and bullet lists with clearer copy.
- **Block schema v1:** Bold, italic, and inline code in the lead editor; marks round-trip to block JSON and PDF export.
- **Typst PDF export:** Worker compiles Markdown via Typst directly (Pandoc removed from worker image).
- **SSE collaboration:** Document, draft, suggestions, and reviews queries poll only when the live-events connection is unavailable.

### Fixes

- Stale suggestion handling (409) after lead draft updates remains enforced server-side.
- Publish still supersedes all pending suggestions in the same transaction.

### Performance

- Smaller worker Docker image without Pandoc.
- Reduced background polling when SSE is connected.
